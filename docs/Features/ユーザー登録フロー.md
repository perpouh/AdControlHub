# ユーザー登録フロー

## アーキテクチャ

 - CloudFront+S3静的ホストでフロントを表示
 - API GatewayでLambda呼び出し
 - LambdaからDynamoDBにユーザーアドレスを書き込み
 - LambdaからSESを呼び出してユーザーにOPT付きメールを送信
 - ユーザーが入力したOTPとDynamoに保存されているOTPを比較、合致したらexample.comへリダイレクト

## ローカル構成

 - S3相当のフロントエンド静的ホストコンテナ（Nginx）
 - API Gateway相当のRubyコンテナ（Sinatra）
 - Lambda相当のPythonコンテナ
 - SES相当のSMTPコンテナ（MailHog）

 - containers直下にそれぞれのディレクトリを作成し、Dockerfileやnginx.conf.template、ソースコードはディレクトリ下に設置すること
 - コンテナ名はわかりやすいものにすること

 ## フロントエンド機能

  - ユーザーはメールアドレスを入力する
  - 入力値がメールアドレスの形式でなかった場合はバリデーションエラーを表示する
  - 入力値がメールアドレスの形式である場合はAPI Gatewayを呼び出し、OTP入力画面を表示する
  - ユーザーはOPTを入力する
  - OTPが合致した場合はexample.comにリダイレクト
  - OTPが合致しない場合は再入力を促し、三回間違えたらメールアドレスの入力からやり直すよう促す

## Lambda機能

 - API Gatewayからユーザーの入力値を受け取り、DynamoDBにメールアドレスとOTPを登録する
   - TTLは24時間とし、期限内に本登録手続きが行われなかった場合はデータが削除されるようにする
   - OTPで使う文字は"ABCDEFGHJKLMNPQRSTUVWXYZ23456789"のみとする
